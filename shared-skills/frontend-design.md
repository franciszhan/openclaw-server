# Frontend Design

Use this skill when building or revising product UI.

## Design Direction

- Pick a clear visual point of view instead of defaulting to generic app UI.
- Use typography intentionally.
- Define a small set of color and spacing variables early.
- Favor strong hierarchy and obvious interaction states.

## Interaction Rules

- Make desktop and mobile layouts both feel intentional.
- Use motion sparingly and meaningfully.
- Avoid decorative animation that does not help comprehension.
- Design empty, loading, and error states on purpose.

## Accessibility Baseline

- Preserve text contrast.
- Keep focus states obvious.
- Do not hide important information behind hover-only behavior.
- Use semantic structure and accessible labels.

## Review Checklist

- Is the layout understandable at a glance?
- Does the page still work on small screens?
- Are typography and spacing consistent?
- Do components feel like they belong to the same system?
- Is the interface distinctive enough to avoid looking boilerplate?
